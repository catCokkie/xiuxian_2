# 05 界面风格规范（参考图）

## 总体方向
- 视觉关键词：古卷、木框、暖棕、像素灵宠。
- 布局结构：`底部常驻条 + 中上部可展开主面板`。
- 目标：既有桌宠陪伴感，也有 RPG 面板的系统深度感。

## 分层定义
1. 桌面底栏（常驻）
- 高度建议：屏幕高的 10%-14%。
- 内容：灵宠动画、境界进度条、AP 实时速率、快捷按钮。
- 交互：支持拖动排序，默认半透明低打扰。

2. 主系统面板（展开）
- 采用“书页/卷轴”形态，顶部页签分为左右两组：
- 左组（游戏内容）：`修炼概况`、`装备情况`、`统计概览`。
- 右组（功能设置）：`联机`、`Bug反馈`、`设置`。
- 主内容区按当前页签切换内容，使用单页全宽展示（不做左右双栏）。
- 左上角提供显式关闭按钮。
- 底部状态条固定显示灵石数量与货币图标。
- 关闭时回到底栏，不销毁状态。

3. 文案管理
- UI 文字统一收口在 `scripts/ui/UiText.cs`。
- 场景中的文本仅作默认值，运行时由控制器从 `UiText` 覆盖。
- 新增文案时优先加到 `UiText`，避免在多个脚本重复硬编码。

## 美术与动效约束
- 主色：羊皮纸米黄 + 木质棕 + 铜金点缀。
- 字体：像素或手写风标题，正文保持可读性。
- 动效：
- 展开面板用“翻页/滑出”0.2-0.3s。
- 底栏灵宠保持低频呼吸与眨眼动画。
- 状态变化（突破/掉落）给短时高亮，不长时间闪烁。

## 可用性约束
- 任何分辨率下底栏不得遮挡任务栏关键区域。
- 面板默认居中，支持记忆上次位置。
- 提供“工作模式”：隐藏面板，仅保留极简底栏。
