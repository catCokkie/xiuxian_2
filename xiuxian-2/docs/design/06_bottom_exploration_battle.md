# 06 底部探索/战斗进度界面详细设计

## 0. 设计目标
- 目标一句话：让玩家在桌面底部用最低打扰成本持续看到“探索推进与战斗结果”。
- 3 秒可读信息：当前区域、探索进度、当前战斗状态、修炼阶段。
- 与键鼠驱动连接：键鼠输入事件直接推进探索条与遭遇计量条；`activity_point` 用于资源结算。

## 1. 双窗口结构
1. 横向主窗口（常驻底部）
- 作用：展示探索与战斗过程。
- 位置：桌面底部贴边，默认横向铺开。
- 特征：低高度、长条形、像素场景演出。

2. 子菜单窗口（图书窗口）
- 作用：以书页/卷轴形式展示系统菜单，分左右两组页签。
- 触发：主窗口左下角“图书按钮”。
- 行为：独立显示/隐藏，不影响主窗口探索计时。

## 1.1 图书窗口结构（固定）
- 窗口样式：摊开书本或卷轴展开，顶部一排页签，底部状态条。
- 顶部分组：
- 左侧页签组（游戏内容）：
- `修炼概况`
- `装备情况`
- `统计概览`
- 右侧页签组（功能/设置）：
- `联机`
- `Bug反馈`
- `设置`
- 底部信息条：
- 固定显示 `灵石数量`，右对齐，带货币图标。
- 可选显示次级货币（V1 可不做）。

## 2. 横向主窗口布局（从左到右）
- 左上：`拖动按钮A`
- 用途：拖动主窗口位置（默认仅限桌面底部水平移动；调试模式可自由移动）。
- 左下：`图书按钮`
- 用途：打开/关闭子菜单窗口。
- 图书按钮下方：`修炼阶段`
- 显示：如“炼气三层/筑基初期”。
- 中央横向区域：`探索战斗演出带`
- 显示：角色、敌人、小怪、地形、遭遇提示。
- 右下：`当前区域探索进度条`
- 显示：区域名 + 0-100% 进度。
- 右上：`拖动按钮B（缩放柄）`
- 用途：调整横向主窗口宽度（最小/最大宽限制）。

## 3. 交互定义
- `拖动按钮A`：
- 鼠标按下后进入拖动态，释放时保存窗口位置。
- `图书按钮`：
- 单击切换子菜单窗口显隐。
- 打开时默认落在上次页签；首次打开默认 `修炼概况`。
- `拖动按钮B`：
- 水平方向拖拽修改主窗口宽度。
- 实时重排内部 UI，最右探索条保持可见。

## 3.1 图书页签交互
- 左右页签组互斥选中，同一时刻仅激活一个页签。
- 切换页签时播放 0.2 秒翻页/滑页动效。
- `Bug反馈` 页签进入后提供：
- 问题描述输入框（多行）
- 一键复制日志路径按钮
- 提交占位按钮（V1 可仅本地导出）
- `联机` 页签在 V1 可显示“开发中”占位，但入口保留。

## 4. 探索与战斗循环
1. 平时状态：`Exploring`
- 键鼠行为持续转换为探索推进值。
- 进度条增长并周期性触发遭遇判定。

2. 遭遇触发：`EncounterWarning -> InBattle`
- 主窗口中央区域切换战斗演出。
- 底部右下进度条暂停或降速（由配置决定，V1 建议暂停）。

3. 战斗结束：`BattleResult`
- 显示胜负、掉落与短结算。
- 返回 `Exploring` 继续推进本区域。

4. 区域完成：`ExploreComplete`
- 进度达到 100%，结算区域奖励并切换下一区域。

## 5. 显示字段与刷新频率
- `realm_stage_text`：修炼阶段，事件触发刷新。
- `zone_name`：当前区域名，区域切换刷新。
- `explore_progress`：0-100%，每 0.2 秒刷新。
- `battle_state`：未遭遇/预警/战斗中/结算中，状态变化刷新。
- `activity_rate`（可选小字）：每秒 AP，1 秒刷新。

## 6. 状态机
1. `Idle`
- 进入：程序启动未开始探索。
- 退出：点击开始探索或自动恢复上次状态。

2. `Exploring`
- 进入：开始探索或战斗结算后。
- 可见：全底栏 + 探索条。
- 退出：遭遇触发或区域完成。

3. `EncounterWarning`
- 进入：遭遇判定成功。
- 可见：中央区域闪烁提示 0.5-1.0 秒。
- 退出：进入 `InBattle`。

4. `InBattle`
- 进入：预警结束。
- 可见：敌我血条、攻击动画、结果倾向。
- 退出：胜负已定。

5. `BattleResult`
- 进入：战斗结束。
- 可见：掉落与奖励飘字 1.2 秒。
- 退出：回 `Exploring`。

6. `ExploreComplete`
- 进入：当前区域探索 100%。
- 可见：区域完成提示与奖励。
- 退出：切下一区域并回 `Exploring`。

## 7. 数值口径（V1）
- 每 100 次键鼠操作 -> 探索进度约 `2.0%`（基础区）。
- `explore_progress_gain = input_events * 0.02% * zone_speed_factor * battle_pause_factor`
- 遭遇判定：每累计 `20%` 探索进度执行一次基础判定。
- 基础遭遇概率：`18% + danger_level * 4%`。
- 单次战斗平均收益：
- `lingqi` 20-60
- `insight` 3-12
- 材料掉落 0-2 个

## 10. 验收标准
- 可以通过左上按钮稳定拖动窗口位置。
- 可以通过右上按钮调整横向窗口宽度且不破版。
- 图书按钮可稳定开关子菜单窗口。
- 右下探索进度条能随键鼠活动持续推进。
- 战斗触发与探索状态切换无卡死。
